# 取引先請求書（売上側）— 動画分析 と 実装ロードマップ

出典: ユーザー提供の画面録画（freee請求書アプリ, 実際に取引先へ送付した5月分請求書）＋ 解析PDF。
`docs_client_invoice_spec.md`（PDF参考仕様）と合わせて読むこと。

## 1. 動画から分かったこと

- 現在の作成ツールは **freee請求書アプリ**（外部SaaS）。この機能を自社システムに取り込みたい。
- 実物（株式会社平安電工宛・5月分）の構造と数字を解読・**検算済み**:
  - 現場名を見出し行に、その下に **電気工事業 A/B/C**（日数×請求単価）＋**交通費**（税率0%）＋**残業代**（時間×単価）。
  - A/B/C は固定額ではなく**プロジェクトごとの請求単価グループ**（読売: A=25,000 / B=21,000 / C=18,000、旧東京現場: A=21,000 …）。同単価の作業員を1行に合算、作業員名は出さない。
  - 残業代 例: 3,906円/時 = **25,000 ÷ 8 × 1.25**（日勤単価から係数1.25で算出）。
  - 税: 10%対象 927,530 → 消費税92,753 / 0%対象（交通費）37,510 → 0 / **小計965,040＋税92,753＝合計1,057,793**。
- freeeで使える編集機能（＝今後ほしい機能）: 取引先/敬称、自社情報・振込先（「この帳票だけ」⇔テンプレ）、社印/ロゴ（ON-OFF・縦位置・倍率・不透明度）、明細設定（取引日表示/消費税端数/金額端数/源泉）、行ごと 摘要・数量・単位・単価・税率、ドラッグ並替・スワイプ削除、小計/税率別内訳/合計 自動、備考＋社内メモ、プレビュー→PDF→送付。

### 追加要件（ユーザー指示・動画外）
- 取引先へは **請求書PDF＋プロジェクト別の交通費領収書＋プロジェクト別の出面表**（誰がいつどの現場か）をセットで渡す → 同梱・添付できるように。
- **会社ロゴは自社→取引先の請求書のときだけ**（作業員が他社へ使い回すのを防止）。作業員向け帳票には会社ロゴを出さない。
- 管理者がアップした画像を、出力する**出面表・請求書PDFの透かし（ウォーターマーク）**として使えるように。

## 2. 重要な発見：機能はすでに半分存在する

- テーブル: `invoices` / `invoice_items`(行ごと税率 10/8/0) / `invoice_supporting_documents`(添付)。
- `company_profile` に `logoUrl` / `sealUrl` / **`watermarkUrl`** / `logoSettings`(JSON) / `sealSettings`(JSON) が**既にある**。
- サーバ: `invoice` ルーター一式（list/get/createFromAttendance/generatePdf/review/approve/return/preview/export…）、`server/pdfInvoice.ts`（**税率別内訳をitemから再計算**）、`server/pdfRoster.ts` に `drawWatermark` が**既にある**。
- クライアント: `AppInvoices.tsx`（1666行のエディタ）に 行ごと税率・テキスト/通常行・並び替え・プレビュー・源泉・敬称 が揃っている。

### 本当のギャップ（生成ロジック側）
1. 生成が **V1締め**（`project_closings`/`closing_submissions`）依存。今月の締めは **月締めV2**で行うが、V2はV1テーブルを更新しないため V2締め現場が請求書生成に出てこない。
2. **交通費0%行が出ない**（取引先に請求漏れ）。
3. **残業代行が出ない**。
4. 社印/ロゴ位置・ウォーターマーク・枝番のUI/PDF反映が未配線（データ構造は存在）。

## 3. PRロードマップ（小さく分割・締め直結順）

- **PR-1（実装済み・締め直結）**: 取引先請求書の生成を**月締めV2軸に繋ぎ替え**＋交通費0%行（現場ごと1行）＋残業代行＋税率別集計。詳細は §4。
- **PR-2**: プロジェクト別**出面表**（作業員×日付×現場）の生成 → 請求書に同梱エクスポート。
- **PR-3**: 社印/ロゴ位置（**自社請求書のみ**・位置/倍率/不透明度）＋**ウォーターマーク**（アップ画像を出面表・請求書PDFに重畳）＋枝番UI。
- **PR-4**: 交通費**領収書**のプロジェクト単位添付（`invoice_supporting_documents`活用）。

### 決定済みの方針
- 単位 = 行ごとに選択可（既定: 労務「日」/ 残業「時間」/ 交通費「式」）。`invoice_items.quantity` の既存規約に合わせ、**単位「日」のみ ×10 保存**（140=14.0日）、他単位は素の値。
- 交通費 = 現場ごとに1行（プロジェクト単位集計・0%税）。
- 源泉徴収は**取引先請求書には載せない**（支払い側＝作業員請求書の話）。`withholdingAmount` は常に0。

## 4. PR-1 実装内容

- `server/clientInvoiceV2Core.ts` — 純粋計算（DB非依存）。`computeClientInvoiceDraft()`:
  現場ごとに 見出し → (請求単価×シフト)でA/B/Cバケット → 残業代（日勤単価÷8×係数, 既定1.25, 必ず要確認警告）→ 交通費(0%, 1行)。
  税は **行ごとtaxRateで税率別集計**（`recalcInvoiceTotals`/`pdfInvoice`と一致）。作業員名は**社内メモのみ**（請求書非表示）。
- `server/clientInvoiceV2Builder.ts` — `buildClientInvoiceDraftFromV2()`:
  ゲート＝V2プロジェクトレビュー「締め完了」優先、無ければV1締め(ready/closed/locked)に**自動ブリッジ**（警告付き）。
  対象者＝V2参加者レビュー（workerIdあり・集計除外でない・確認済み/締め完了。ゲスト除外）、無ければ在籍メンバー＋出面にフォールバック。
  交通費＝`getMonthlyClosingV2ClientTransportationBillingSummary`（isClientBillableのみ・客先直払除外）。
  単価未設定は**ハードエラーにせず ¥0行＋警告**（編集画面で単価設定→再生成）。
- `server/routers.ts` — `closing.generateForClosing` / `invoice.generateForClosing` / `invoice.createFromAttendance` の3か所をV2ビルダーに移行。戻り値に `warnings` を追加。
- `client/src/pages/AppClosings.tsx` — 生成時の要確認事項を、編集画面に遷移しても残る警告トーストで表示。
- テスト: `server/clientInvoiceV2Core.test.ts`（A/B/C・夜勤別行・残業導出・単価未設定・**実物5月請求書の合計 965,040/92,753/1,057,793 を再現**）。`closing.consolidated-invoice.test.ts` をV2軸へ更新。

### PR-1の既知の限界（後続で扱う）
- 残業単価は日勤単価×係数(既定1.25)からの**自動推定**（必ず警告）。契約別の残業単価は将来 `employee_rates` 等の設定化で精緻化。
- 自社負担/客先直払/対象外/ゲストの交通費除外は V2集計側で担保（PR-4で領収書添付と合わせ再点検）。
- 社印/ロゴ/ウォーターマーク/枝番のPDF反映は PR-3。
