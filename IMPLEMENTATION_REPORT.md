# IMPLEMENTATION REPORT (2026-04-30)

## 重要修正: AppRates 編集・削除の復旧
- すべての単価レコードに対し、デスクトップのテーブルとモバイルのカード一覧の双方で**編集**・**削除**アクションを復旧。
- モバイルカードに明確でタップ可能なアクションボタンを追加:
  - `編集`
  - `削除`
- モバイルの編集モードはインラインかつ縦積みにし、スマホ画面での見切れ／溢れを防止。
- 削除アクションは既存の `trpc.rate.delete` を使用し、実行前に確認ダイアログを表示。
- 編集保存は既存の `trpc.rate.update` を使用。

## データ安全性・バリデーション挙動
- 編集値の安全な数値パースを追加し、空欄は `0` への強制変換ではなく未入力（`undefined`）として扱う。
- 編集時のバリデーションを強制:
  - 両方空（売上単価・支払単価）→ 却下
  - 売上単価のみ → 許可
  - 支払単価のみ → 許可
  - 両方あり → 許可
- 既存の単価スコープ（プロジェクト／取引先スコープ）の挙動は維持。
- 売上／支払の分離を維持（双方入力の強制なし）。
- 更新ペイロードで「0円フォールバックを有効な未設定単価とみなす」ことはしない。

## 明示的な非変更
- スコープ・アーキテクチャのロールバック／リバートなし。
- 売上／支払分離のロールバック／リバートなし。
- API エンドポイントの置き換えなし（既存の trpc ミューテーションを再利用）。
