# Beta検証データ（固定Betaセット / テストフィクスチャ）

最終更新: 2026-06-28

検証・テストを**本番データに触れず・再現可能**に行うための固定データセットです。
（ソフトウェア開発でいう「テストフィクスチャ／シードデータ」）

## 固定Betaセット（一度作って使い回す）

| 種別 | 名前 | 備考 |
|---|---|---|
| 取引先 | `Beta_Client_01` | Beta検証用 |
| 作業員 | `Beta_Worker_01` | Beta検証用 |
| 現場 | `Beta_Project_01` | `Beta_Client_01` に紐付け |
| テスト月 | **2024-01 固定** | 2025年以前 ＝ 本番データと混ざらない |

投入される内容（作業員請求書V2＋日報を端から端まで検証できる）:
- 出面（昼勤13日＋夜勤2日、うち2日に残業）
- 作業員単価（昼 ¥18,000 / 夜 ¥22,000）
- 提出済みの締め（V1）＋交通費 ¥24,000・経費 ¥3,000 → V2ビルダーがブリッジして請求書生成

## 使い方（super_admin）

1. super_admin でログイン
2. `/app/worker-invoice-v2`（作業員請求書V2）を開く
3. 右上の「**Beta検証データを作成/リセット (2024-01)**」ボタンを押す
4. 自動で `Beta_Worker_01` × `2024-01` が選択され、請求書＋日報が表示される

ボタンは **super_admin のみ**表示。サーバ側も `betaFixture.seed` を `superAdminProcedure` で保護。

## ルール（MANUS等への標準指示）

```
テスト・検証時のルール:
① 既存のBetaプロフィール（Beta_Worker_01、Beta_Client_01）のみ使用
② 固定テスト月は 2024-01
③ 新規プロフィール・レコードの作成は禁止（既存Betaを上書き編集する）
④ Betaデータの編集・上書き・数値入力（出面・残業時間・単価等）は自由に行ってよい
⑤ 本番データには一切触れないこと
```

## 安全性

- シードは **`Beta_*` エンティティ（名前で検索/作成）と 2024-01 だけ**に触れる。本番データは触れない。
- **冪等**：何度実行しても重複プロフィールを作らず、Beta_Project_01 × Beta_Worker_01 の単価と 2024-01 の出面・締めをベースラインにリセットする。
- 実装: `server/betaFixture.ts`（テスト: `server/betaFixture.test.ts`）。

## メリット
- データベースが汚れない / テスト結果が再現可能 / 本番データと混ざるリスクゼロ
