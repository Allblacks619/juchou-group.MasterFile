# 従業員管理システム是正メモ 現コード調査（Phase 4B-1）

調査日: 2026-05-06
対象: 従業員管理・権限・一括操作・スマホUIの現コード確認

## 結論

是正メモ相当の項目は一部改善済みですが、全体を一気に実装するには削除影響範囲の洗い出し、二重確認UI、監査ログ、テスト追加が必要です。Phase 4B-1では既存機能を壊さないことを優先し、画面レイアウトと権限表示の小さな修正に限定しました。

## 改善済み・部分対応・未対応の整理

| 項目                            | 現状       | 根拠                                                                                                                                                               | 今回対応                                |
| ------------------------------- | ---------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------ | --------------------------------------- |
| super_admin専用の一括ロール変更 | 部分対応   | `superAdmin.bulkChangeRoles` は `superAdminProcedure` 配下で、`super_admin` への一括昇格も拒否しています。ただしUI側は影響プレビューを使っていません。             | 変更なし（大きい修正候補）              |

## 今回の小修正

1. 従業員一覧をスマホではカード表示、タブレット以上では既存テーブル表示に分岐しました。
   - チェックボックス、氏名、フリガナ、国籍、電話番号、詳細遷移をカード内に収め、横幅による崩れを避けます。
   - 既存テーブルのデスクトップ操作は維持しています。
2. 選択中バーを折り返し可能にして、スマホで「選択解除」ボタンがはみ出しにくくしました。
3. 従業員詳細の管理者判定に `super_admin` と `manager` を含め、現行APIの管理者系権限とUI表示の差を小さくしました。

## 次PRへ分けるべき大きい修正

- 単体削除を super_admin 専用に寄せるか、少なくとも対象super_admin保護を追加する。
- 一括削除前の影響プレビューを実データで返す（出面、現場メンバー、締め、支払、請求明細、ユーザー連携、監査ログなど）。
- 一括削除UIをプレビュー付き二重確認ダイアログに変更する。
- 一括/単体のロール変更・削除に監査ログを追加する。
- 上記について roleAccess 系テストを追加・拡張する。

## Phase 4B-1であえて触らなかった理由

従業員削除やロール変更はデータ消失・権限昇降格に関わるため、影響範囲確認とテストなしに大きく変えると既存運用を壊すリスクがあります。今回はUI崩れ防止と権限表示漏れの修正に留め、削除安全性の本格対応は次PRで実装するのが安全です。
